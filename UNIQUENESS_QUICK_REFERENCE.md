# Quick Reference: Department Uniqueness Changes

## 🎯 What Changed

Department uniqueness is now **per-institute** instead of **global**.

---

## 📦 Files Modified

### 1. DepartmentRepository.java
**Added:** New query method
```java
boolean existsByNameAndInstituteId(String name, Long instituteId);
```

### 2. DepartmentService.java
**Updated:** `createDepartment()` method
**Updated:** `updateDepartment()` method

---

## ✅ Validation Rules (Updated)

| Rule | Before | After |
|------|--------|-------|
| **"CS" at MIT** | Globally unique | ✅ Allowed |
| **"CS" at Stanford** | ❌ Blocked (duplicate) | ✅ Allowed |
| **"CS" at MIT (2nd time)** | ❌ Blocked | ❌ Still blocked |

---

## 🔍 How It Works

### Create Department
```
Check: existsByNameAndInstituteId(name, instituteId)
If exists: ❌ Block with error
If not: ✅ Allow creation
```

### Update Department
```
If name changed:
  Check: existsByNameAndInstituteId(newName, instituteId)
  If exists: ❌ Block
  If not: ✅ Allow
If institute changed:
  Check against new institute
```

## 💡 Key Features

✅ Same name in different institutes = **ALLOWED**  
✅ Same name in same institute = **BLOCKED**  
✅ Null-safe institute ID handling  
✅ Clear error messages  
✅ No database changes needed

## 📡 API Examples

### ✅ Success: Create "CS" at different institutes
```bash
# At MIT (institute_id = 1)
curl -X POST /api/departments \
  -d '{"name": "CS", "instituteId": 1}'
# Response: 201 Created ✅

# At Stanford (institute_id = 2)
curl -X POST /api/departments \
  -d '{"name": "CS", "instituteId": 2}'
# Response: 201 Created ✅ (allowed, different institute)
```

### ❌ Error: Duplicate in same institute
```bash
curl -X POST /api/departments \
  -d '{"name": "CS", "instituteId": 1}'
# Response: 400 Bad Request
# Message: "Department with name 'CS' already exists in this institute"
```
